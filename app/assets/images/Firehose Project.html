We will adjust the footer to have 4 columns of links, unless it's the width of a mobile phone. If the width is the size of a mobile device, we will display the separate columns on top of each other.


Let's get coding!

Add a link to our user dashboard to the footer of our application underneath the Teach a Course link. To find the destination of the link, look up the value in the routes table. Run the following command to output the contents of the routes table. 

$ rake routes

We want to send the user to the user dashboard page, or in other words the dashboards controller and the show action. The line in the routes table that represents that looks like this: 

dashboard GET    /dashboard(.:format)                               dashboards#show

This means we're going to send the user to the dashboard_path. Edit app/views/layouts/application.html.erb to add the link to the footer. 

  <div id="footer">
    <div class="container">
      <b>Teachers</b>
      <br />
      <%= link_to 'Teach a Course', new_instructor_course_path %>

      <br />
      <%= link_to 'My Dashboard', dashboard_path %>
    </div>
  </div>

Save the file and refresh the page. Notice how the link to the user dashboard appears in the footer.

There are many different links to add to the footer. We also know we want to setup our footer with 4 columns. Since our footer will have a lot of content, rather than showing you what to write line-by-line, let's add a placeholder for each section in one fell swoop. Add the following to the footer in app/views/layouts/application.html.erb: 

  <div id="footer">
    <div class="container">

      <div class="col-xs-12 col-sm-3">
        <b>Teachers</b>
        <br />
        <%= link_to 'Teach a Course', new_instructor_course_path %>
        <br />
        <%= link_to 'My Dashboard', dashboard_path %>
      </div>
      <div class="col-xs-12 col-sm-3">
        <b>About</b>
        <br />
        <%= link_to 'Team', '#' %>
        <br />
        <%= link_to 'Careers', '#' %>
        <br />
        <%= link_to 'Privacy Policy', '#' %>
      </div>
      <div class="col-xs-12 col-sm-3">
        <b>Contact</b>
        <br />
        ken@thefirehoseproject.com
        <br />
        One Kendall Square
        <br />
        Cambridge MA, 02113
      </div>
      <div class="col-xs-12 col-sm-3">
        <b>Social</b>
      </div>
    </div>
  </div>

Save the file and refresh the page. Notice how a lot of content was added to the footer of the application.

The text that is present within the footer doesn't look great, because we haven't tweaked the properties of the fonts that we're using. Add a font-weight of 100 for non-links that are within the footer by adding this line in app/assets/stylesheets/footer.css.scss. 

/* Set the fixed height of the footer here */
#footer {
  height:$footer-height;
  color:white;
  background-color: rgba(36,32,33,0.9);
  font-family: "Helvetica Neue",Helvetica;
  padding:10px 0px;

  font-weight:100;
}

Save the file and refresh the page. Notice the impact the font change has.

The links to social media websites we intend to add to the "Social" column would look better if we used icons rather than text. Visit IconFinder.com and download a few icons into the app/assets/images folder.

I want to link to Facebook, GitHub and Twitter, so I downloaded images into my application with the file names github.png, facebook.png and twitter.png. Feel free to add different links in your application and change the icons accordingly.

Once you have the files in your web application add the images to the footer by adding the following code to app/views/layouts/application.html.erb: 

      <div class="col-xs-12 col-sm-3">
        <b>Social</b>

        <%= image_tag 'github.png' %>
        <%= image_tag 'facebook.png' %>
        <%= image_tag 'twitter.png' %>
      </div>

Save the file and refresh the page. Notice how the social media logos appear in the footer of our application.

The images appear bigger than I'd like. It would be easy to change the size of them if they had a custom class that we could write a CSS rule for. Edit app/views/layouts/application.html.erb to add a social-button class to the items: 

      <div class="col-xs-12 col-sm-3">
        <b>Social</b>

        <%= image_tag 'github.png', class: 'social-button' %>
        <%= image_tag 'facebook.png', class: 'social-button' %>
        <%= image_tag 'twitter.png', class: 'social-button' %>
      </div>

Save the file.

Adjust the size of the images that have this property to be 32 pixels wide in app/assets/stylesheets/footer.css.scss. 

.social-button {
  width: 32px;
}

Save the file and refresh the page. Notice how the icons got smaller.

Next, let's move the social icons lower on the page by adding a few breakline tags in app/views/layouts/application.html.erb: 

      <div class="col-xs-12 col-sm-3">
        <b>Social</b>

        <br />
        <br />
        <%= image_tag 'github.png', class: 'social-button' %>
        <%= image_tag 'facebook.png', class: 'social-button' %>
        <%= image_tag 'twitter.png', class: 'social-button' %>
      </div>

Save the file and refresh the page.

Add a little bit of space between the icons, by adding a margin of 4 pixels to the image in app/stylesheets/footer.css.scss. 

.social-button {
  width: 32px;

  margin:4px;
}

Save the file and refresh the page. The footer is starting to look good, but the social icons links are not functional.

To start, let's hook up the GitHub link. I want to link the icon to my GitHub handle, so the URL looks something like http://github.com/kenmazaika or http://github.com/marcomorawec. You'll want to adjust the link to go to your profile page on the social media site. Edit app/views/layouts/application.html.erb to look something like this: 

      <div class="col-xs-12 col-sm-3">
        <b>Social</b>
        <br />
        <br />

        <%= link_to 'http://github.com/kenmazaika' do %>
          <%= image_tag 'github.png', class: 'social-button' %>
        <% end %>
        <%= image_tag 'facebook.png', class: 'social-button' %>
        <%= image_tag 'twitter.png', class: 'social-button' %>
      </div>

Save the file and refresh the page.

Follow the same pattern for each of the different social media sites you're linking to. In our case, we would hook up the links to Facebook and Twitter as well.

Notice how when you hover over the social media icon an underline appears and it looks a little off. By default links have a text-decoration of underline, which we should disable. In order to add a new CSS property, add a CSS class to the div that contains the social buttons. Edit app/views/layouts/application.html.erb to look like this: 

      <div class="col-xs-12 col-sm-3 social">
        <b>Social</b>
        <br />
        <br />
        <%= link_to 'http://github.com/FirehoseWeekend' do %>
          <%= image_tag 'github.png', class: 'social-button' %>
        <% end %>
        <%= image_tag 'facebook.png', class: 'social-button' %>
        <%= image_tag 'twitter.png', class: 'social-button' %>
      </div>

Save the file.

Disable the text-decoration for social links on hover, by adding the following rule to app/stylesheets/footer.css.scss: 

.social a:hover {
  text-decoration:none;
}

Save the file and refresh the page. It looks good!

Media Queries

The footer usually looks good, but if we view the page in the extra-small size, the height of the footer is not large enough for the content that is within it.

We have set the height of the footer to be exactly 100 pixels in footer.css.scss, which makes sense if we have 4 separate columns. In extra-small size, we only have a single column, so it would be good to allow the height of the footer to be double that.

We can override a CSS property only on certain sized screens by using what is called a media query. Add the following code to double the height of the footer if the width is less than 750 pixels, the size of the largest screen in extra-small mode. Edit app/stylesheets/footer.css.scss to include the following code at the bottom of the file. 

$footer-height-mobile: 200px;
@media (max-width: 750px) {
  #footer {
    height:$footer-height-mobile;
  }

  #wrap {
    margin: 0 auto -$footer-height-mobile;
    /* Pad bottom by footer height */
    padding: 0 0 $footer-height-mobile;
  }

}

Save the file and refresh the page. When the page loads, adjust the width of the page and pay attention to how the size of the footer changes depending on the mode the page is displayed in.

Within this same media query, add a CSS rule that will push the "Social" header further down the footer. Add the following lines to app/stylesheets/footer.css.scss: 

$footer-height-mobile: 200px;
@media (max-width: 750px) {
  #footer {
    height:$footer-height-mobile;
  }

  #wrap {
    margin: 0 auto -$footer-height-mobile;
    /* Pad bottom by footer height */
    padding: 0 0 $footer-height-mobile;
  }


  #footer .social {
    margin-top:10px;
  }

}

Save the file and refresh the page. Take a moment to appreciate how your footer looks.

How to get a fast answer

How to get a fast answer for your question:

Heres how to ask smart questions in the forum:


First, give it a good title that describes what youre struggling with. Descriptive headlines like my quotes dont show on my index page or the email notification email is not triggered at comment submission work best.


Second, describe in detail what youve done up to this point. For example, you should say things like: I added a quote through the rails console and can see it in my database. I also added the code to create the loop on my index.html.erb page and added the necessary code inside of my quotes controller too.


Third, describe in detail what you expected to happen. For example, you should describe things like this: I refreshed my browser window with the url localhost:3030 and expected a new quote to be shown on my root page.


Fourth, paste a screenshot of your error message. This one is obvious, if we can see what youre seeing we can help you a lot better. Make sure you get the whole page.


Fifth, include a link to your github repository: If we cant figure out how to help you through your description above, we often times go directly into GitHub and take a look at your web application code, pull it onto our machines, and give it a little test drive.

Important: Make sure you always commit and push the latest version of your code to your GitHub repository, before submitting your question in the Q&A forum. By doing so we can look at your latest code and see the same error message as you.


Sixth, include a link to you web application: We might need to have a look at your app live on the internet. Including the url for your app will help. Plus if you don't include the url, you might get a response from Jarvis, the firehose robot, requesting that you add it.


Speed of help: We generally check out forum several times each day and answer all questions. The better you formulate your problem, the faster you get your solution.

This makes sense. If you forget to include a screenshot of your error message, the most likely reply that you get back will be something like hey there, can you please include a screenshot of your error message so we can see what youre seeing?.


Directly emailing your Mentor a question: The forum is a lot faster way to get help - since we always have somebody on dedicated forum watch. It might seem tempting to go around the forum and email your mentor directly, but that will most likely get you a delayed response, since your mentor might be helping other students and will not get back to you as fast.
